# API interfaces use methods

## The preparatory work

All interface calls are made through the rayvision_api module, and an API object must be instantiated before use :

```python user_info = {

“domain_name”: “jop.foxrenderfarm.com”, “platform”: “2”, “access_id”: “xxxxxxxxxxxxxxxxxxxxxx”, “access_key”: “xxxxxxxxxxxxxxxxxxxxx”,

}

api = RayvisionAPI(access_id=user_info[‘access_id’],

access_key=user_info[‘access_key’], domain=user_info[‘domain_name’], platform=user_info[‘platform’])

```

Warning：

> 1. the following interface calls will be made directly using the API of the above instance; > 2. In the rayvision_api, the interface actually returns the user only the value of the “data” parameter ;

Interface path：/api/render/hardwareConfig/list

Request parameter：

<table border=”1” class=”docutils”> <thead> <tr> <th><strong>Parameter</strong></th> <th><strong>Type</strong></th> <th>Necessary</th> <th><strong>Description</strong></th> </tr> </thead> <tbody> <tr> <td>task_ids</td> <td>List[str]</td> <td>N</td> <td>A collection of task numbers that queries the configuration parameters of the specified task</td> </tr> </tbody> </table>

Return parameter[data]：

<table border=”1” class=”docutils”> <thead> <tr> <th><strong>Parameter</strong></th> <th><strong>Type</strong></th> <th><strong>Description</strong></th> </tr> </thead> <tbody> <tr> <td>id</td> <td>int</td> <td>Hardware Configuration ID (HardwareConfigID)</td> </tr> <tr> <td>type</td> <td>int</td> <td>1: CPU ;<br>2: GPU</td> </tr> <tr> <td>model</td> <td>string</td> <td>Hardware model, default “default”</td> </tr> <tr> <td>ram</td> <td>String</td> <td>memory</td> </tr> <tr> <td>gpuNum</td> <td>String</td> <td>Number of GPU cards, CPU platform is “null”</td> </tr> <tr> <td>platform</td> <td>int</td> <td>Platform number</td> </tr> <tr> <td>current</td> <td>bool</td> <td>When task_ids is not null， the query is the task’s current hardware configuration<br>false: is the current task hardware configuration<br>true: Is not the current task hardware configuration.</td> </tr> <tr> <td>notSupportCgId</td> <td>list[int]</td> <td>Unsupported cgId, (cgid corresponding software can query “General Parameter” – &gt; “ID mapping of DCC software”)</td> </tr> <tr> <td>status</td> <td>int</td> <td>Status,<br>1: Enabled;<br>0: Disabled</td> </tr> </tbody> </table>

Example of request：

`json hardware_config = api.user.get_hardware_config(task_ids=["6306543"]) `
